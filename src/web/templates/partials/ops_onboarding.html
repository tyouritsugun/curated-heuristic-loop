How to use this page

Each card mirrors an API endpoint, so everything you do here is logged and protected by the same safety checks.

{% if search_mode == 'sqlite_only' %}
  1. Start with data operations. Use Import to pull from Google Sheets and Export to push local edits out. Each trigger records the actor, lock status, and result in Job History.
  2. Watch queue + telemetry. The SSE stream refreshes queue status and recent jobs without reloading the page. If SSE is blocked, press the "Refresh" buttons on each card.
  3. Search uses keyword matching. In CPU-only mode, entries are immediately available for search after import. Use specific keywords from titles and content. See CPU-Only Mode docs for search tips.
{% else %}
  1. Start with data operations. Use Import to pull from Google Sheets and Export to push local edits out. Embeddings are generated automatically after import. Each trigger records the actor, lock status, and result in Job History.
  2. Manage models as needed. The Model Management card lets you change embedding and reranker models. Model changes will automatically trigger re-embedding of all content (Phase 4).
  3. Watch queue + telemetry. The SSE stream refreshes queue depth, worker heartbeats (when available), and recent jobs without reloading the page. If SSE is blocked, press the "Refresh" buttons on each card.
  4. Handle FAISS snapshots. Use the snapshot card to download the current index or upload a prepared ZIP (stays on disk, logged in audit trail). This is the safest way to move vectors between machines.
{% endif %}
